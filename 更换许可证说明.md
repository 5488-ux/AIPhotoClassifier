# 许可证说明和更换指南

## 当前许可证：MIT License

### 常见开源许可证对比

| 许可证 | 商业使用 | 修改 | 分发 | 专利授权 | 要求开源 |
|--------|---------|------|------|---------|---------|
| **MIT** | ✅ | ✅ | ✅ | ❌ | ❌ (最宽松) |
| **Apache 2.0** | ✅ | ✅ | ✅ | ✅ | ❌ |
| **GPL 3.0** | ✅ | ✅ | ✅ | ✅ | ✅ (必须) |
| **BSD 3-Clause** | ✅ | ✅ | ✅ | ❌ | ❌ |
| **Unlicense** | ✅ | ✅ | ✅ | ❌ | ❌ (公有领域) |

## MIT License（当前）

### 优点
- 最简单、最流行
- 允许商业使用
- 允许闭源使用
- 企业友好

### 缺点
- 无专利保护
- 无商标保护

### 适用场景
- ✅ 个人项目
- ✅ 学习示例
- ✅ 工具库
- ✅ 希望最大化传播

## 如何更换许可证

### 方法1：更换为Apache 2.0（如果需要专利保护）

```bash
cd C:/Users/Admin/AIPhotoClassifier

# 下载Apache 2.0 License
curl -o LICENSE https://www.apache.org/licenses/LICENSE-2.0.txt

# 或手动创建 LICENSE 文件，内容从这里复制：
# https://www.apache.org/licenses/LICENSE-2.0.txt

# 提交更改
git add LICENSE
git commit -m "Change license to Apache 2.0"
git push
```

### 方法2：更换为GPL 3.0（如果要求衍生作品也开源）

```bash
cd C:/Users/Admin/AIPhotoClassifier

# 下载GPL 3.0 License
curl -o LICENSE https://www.gnu.org/licenses/gpl-3.0.txt

# 提交更改
git add LICENSE
git commit -m "Change license to GPL 3.0"
git push
```

### 方法3：移除许可证（不推荐）

```bash
cd C:/Users/Admin/AIPhotoClassifier

# 删除LICENSE文件
git rm LICENSE
git commit -m "Remove license"
git push
```

**注意**：无许可证意味着默认"保留所有权利"，别人不能合法使用。

## 推荐决策树

### 你希望别人如何使用你的代码？

1. **随便用，闭源也行** → **MIT License** ✅（当前）
2. **可以闭源，但要专利保护** → **Apache 2.0**
3. **必须开源分享改进** → **GPL 3.0**
4. **完全放弃版权** → **Unlicense**
5. **不想分享** → **删除LICENSE + 设为Private仓库**

## 常见问题

### Q: 有License后别人可以商用我的代码？
**A**: 是的，MIT允许商业使用。但他们必须保留你的版权声明。

### Q: 我会失去代码所有权吗？
**A**: 不会！你仍然是原作者和版权所有者。

### Q: 我能改主意吗？
**A**: 可以，但已发布的版本仍受原许可证约束。新版本可以换许可证。

### Q: 我必须要License吗？
**A**: 对于公开的GitHub仓库，强烈建议添加。

### Q: MIT License有什么风险？
**A**: 
- 别人可以闭源商用（如果你在意）
- 无专利保护
- 但对大多数项目来说都不是问题
